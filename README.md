# BabyTrip - 宝宝出行助手

一个基于 Swift + SwiftUI 的原生 iOS + watchOS 应用，帮助家长判断当前天气环境是否适合带宝宝外出。

## 版本更新

> **V4 重要升级** - 统一裁决 + 偏好驱动 + 体验优化

- 🧠 **统一评分引擎**：iOS 和 watchOS 共用 `TripAdvisor` 统一裁决服务，消除两端评分不一致问题
- ⚙️ **偏好驱动评分**：用户自定义的温度、紫外线、AQI、风速等偏好阈值真正参与评分计算，修改偏好后结果有明显变化
- 🔒 **SSL 安全加固**：生产环境恢复正常的证书验证，开发环境保留调试灵活性
- 🎨 **首页精简**：核心结论「能不能出门」及其原因首屏直接展示，次要信息合理收纳
- 🌤 **数据源升级**：从 OpenWeatherMap 迁移到 [和风天气 (QWeather)](https://dev.qweather.com/)，国内数据更精准稳定
- 🔔 **天气变化提醒**：当天气变得不适宜外出时主动推送通知提醒

## 功能特性

- 📱 支持 iOS 17+ 和 watchOS 10+ 双平台
- 🌤 根据宝宝月龄自动调整风险敏感度，给出个性化出行建议
- 📍 自动定位获取当前位置天气，支持手动选择城市
- 🧮 **六维综合评估**: 温度 + 体感温度 + 紫外线 + 空气质量 + 风速 + 湿度
- 💡 **智能解读**: 人话总结 + 阻断/警告/正面三层次原因 + 可执行行动建议
- ⏰ **智能推荐今日最佳外出时间段**，分析 24 小时天气预报帮你选对时间出门
- 👔 **穿衣指南**: 根据体感温度、风速、天气状况智能推荐宝宝穿衣搭配
- 📊 **昨日对比**: 今天和昨天天气变化趋势一目了然
- ⚖️ **年龄权重算法**: 0-3个月 ×1.5倍风险权重（更敏感），月龄越小越严格
- 🎨 **偏好驱动**: 自定义温度范围、紫外线阈值、偏好外出时段，阈值直接参与评分计算
- 🔔 **主动推送提醒**: 天气变化不适宜外出时发送本地通知，根据宝宝月龄自动调整提醒阈值
- 🧠 **统一裁决**: `TripAdvisor` 作为唯一评分入口，iOS/watchOS 共用，整合五大评估模块
- 💾 数据全部本地存储，无需登录，保护隐私
- 🔄 下拉刷新快速更新天气数据
- ⚙️ 设置向导，初次使用一键配置宝宝信息

```bash

cd haojian/BabyTrip
```

### 2. 配置 API Key

项目使用 [和风天气 (QWeather)](https://dev.qweather.com/) 免费开发者 API：

1. 注册和风天气开发者账号，创建免费开发版项目
2. 获取你的 API Key (Key)
3. 分别打开 `BabyTrip/Info.plist` 和 `BabyTripWatch/Info.plist`
4. 将 `QWeatherAPIKey` 字段的值 `YOUR_API_KEY_HERE` 替换为你的 API Key

> 免费开发版足够个人使用，每天 1000 次调用限制。

### 3. 编译运行

```bash
open BabyTrip.xcodeproj
```
选择 iOS 17+ 模拟器或真机，点击运行。

更详细的调试步骤和常见问题排查请看 [DEBUG.md](./DEBUG.md)

## 评估规则

### 架构说明 (V4)

BabyTrip 采用**统一裁决架构**，通过 `TripAdvisor` 整合所有评估模块，确保各模块建议一致。iOS 和 watchOS 共用同一套评分逻辑，消除两端结果不一致问题：

```
天气数据 → TripAdvisor (统一裁决)
              ├── 硬过滤检查 (一票否决)
              ├── RiskEvaluator (风险评分，含偏好权重)
              ├── TripRecommendationEngine (时段推荐)
              └── 穿衣指南
              ↓
         统一输出 TripAdvice
         ├── 是否适合外出 (最终裁决)
         ├── 一致的风险等级 (iOS/watchOS 完全一致)
         ├── 偏好匹配检查结果
         └── 统一的行动建议
```

### 偏好驱动评分

用户偏好设置不再只是表面提示，而是直接影响评分计算：

| 偏好项 | 对评分的影响 |
|--------|-------------|
| 理想温度范围 | 超出设定范围时增加风险分 |
| 最大紫外线 | 超过阈值时加权扣分 |
| 最大风速 | 超过阈值时增加风险分 |
| 最大 AQI | 超过阈值时增加风险分 |
| 偏好外出时段 | 推荐时段优先匹配用户习惯 |

提供「恢复默认」一键重置功能，方便用户回到原始权重。

### 硬过滤机制 (一票否决)

以下任一条件触发，直接判定为**不适合外出**，忽略其他评分：

| 条件 | 阈值 | 说明 |
|------|------|------|
| 恶劣天气 | 雨/雪/雷/沙尘/雹/雾/霾 | 关键词匹配 |
| 风速 | ≥36 km/h | 大风天 |
| 空气质量 | AQI ≥ 150 | 重度污染 |
| 温度 | < 0°C | 严寒 |
| 体感温度 | > 38°C | 酷热难耐 |

### 六大环境维度 (软评分)

通过硬过滤后，使用**风险累加评分系统**，总分越高风险越大：

| 因素 | 风险加分规则 | 说明 |
|------|-------------|---------|
| 温度 | >32°C +25 / <5°C +30 | 过高过低都增加风险 |
| 体感温度 | >35°C +20 / <0°C +15 | 反应实际感受 |
| 紫外线 | ≥7 +25 / ≥4 +10 | 强烈紫外线对宝宝伤害大 |
| 空气质量 | >100 +30 / >50 +15 | 空气污染对呼吸道刺激 |
| 风速 | >25km/h +10 | 大风天气需要避风 |
| 湿度 | >90% +15 / >80% +10 / <30% +5 | 太闷太干都不舒服 |
| 天气状况 | 雨/雪/雷/沙/尘/雹/雾/霾 | 恶劣天气直接 +50 分 |

### 年龄权重（风险放大）

| 月龄 | 权重 | 说明 |
|------|------|------|
| 0-3个月 | ×1.5 | 新生儿，更加敏感 |
| 3-6个月 | ×1.2 | 小婴儿，中高敏感 |
| 6-12个月 | ×1.0 | 婴儿，中等敏感 |

### 最终评级

| 风险总分 | 建议等级 |
|---------|---------|
| 0-30 | 🟢 **适合外出** |
| 31-60 | 🟡 **谨慎外出** |
| >60 | 🔴 **不建议外出** |

### 输出内容

V4 不仅给结论，还给你完整的决策辅助：
1. **人话总结**：一句话说清楚今天能不能出门
2. **原因明细**：列出哪些因素拉高了风险（阻断/警告/正面三层次分类）
3. **行动建议**：每个风险点给具体防护建议
4. **偏好匹配**：根据你的偏好检查天气是否符合预期
5. **最佳时段**：推荐今日评分最低的连续时间段适合外出
6. **穿衣指南**：根据体感温度、风速、天气状况智能推荐宝宝穿衣搭配
7. **昨日对比**：与昨天对比天气变化趋势，一目了然

## API 说明

当前使用 [和风天气 (QWeather) API v7](https://dev.qweather.com/docs/api/)：
- **GeoAPI**: 根据经纬度获取城市 ID
- **实时天气 API**: 获取当前温度、体感温度、风速、湿度
- **空气质量 API**: 获取 AQI 指数
- **天气指数 API**: 获取紫外线指数
- **24小时预报 API**: 获取逐小时预报用于计算最佳外出时段
